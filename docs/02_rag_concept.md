## 주요 내용

### 1. RAG 파이프라인

1. **문서 로딩**: 다양한 소스에서 문서 수집
2. **청킹**: 문서를 작은 단위로 분할
3. **임베딩**: 텍스트를 벡터로 변환
4. **저장**: 벡터를 벡터 스토어에 저장
5. **검색**: 쿼리와 유사한 문서 검색
6. **생성**: 검색된 문서를 컨텍스트로 LLM 응답 생성

### 2. RAG의 장점

- 최신 정보 활용 가능
- 도메인 특화 지식 통합
- 환각(Hallucination) 감소
- 출처 추적 가능

### 3. RAG의 한계

- 검색 품질에 의존
- 컨텍스트 윈도우 제한
- 실시간 정보 부족
